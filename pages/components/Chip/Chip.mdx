# Chip

The chip component provides the user the ability to select and filter the represented information in an easy and clear way. 
Chips generally appear as a group of components that interact together (see ChipGroup).

## Anatomy
Chips are made up of three key elements:

- **Container**: This element manages sizing and spacing between these elements.
- **Content**: A label text should be always displayed. It's important to be concise and clear.
- **Icon**: It is optional and helps to categorise the Chip.
- **Action remove**: It is optional and facilitates the removal of the Chips.

### Sizes
We're using usually the 'md' size for our chips, but in some circumstances or contexts we need to use some variants
because of space and design reasons.

| Size &nbsp; &nbsp; &nbsp; &nbsp; | Usage                                                                                                                                                                                                                                    |
| -------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **xxs**                          | It size is only applied to chips in a selected group. |
| **xs**                           | This is the default size and the most common.    |
| **sm**                           | This is the default size and the most common.   |
| **md**                           | This is the default size and the most common.                                                                                                                                                                                               |
| **lg**                           | The expressive size. It is barely used in the App context, only in large or commercial contexts. 

### States
 The dynamic states of the container effectively communicate to the user the various states of the Chips:
<Figure>
  <Figma
    aspectRatio="16x9"
    url="https://www.figma.com/file/4f9wa0fKYIwMue6yo6nrJL/GDS---Web-Previews?page-id=18566%3A46223&node-id=18944-6360&node-type=frame&viewport=173%2C-457%2C1.47&t=E9PUii2tvp83bLE9-1&scaling=min-zoom&content-scaling=fixed"
  />
</Figure>
- **Default**: Represents the appearance of the chip before interaction.
- **Hovered**: When the user interacts with an element with a pointing device, but does not necessarily activate it.
  It is generally triggered when the user hovers over an element with the cursor (mouse pointer). Sometimes we can use
  this state to highlight an action over other ones. The styles bound to this state are equivalent to pseudo-class
  `:hover` ones.
- **Focused**: when the chip component has received focus. It is generally triggered when the user clicks or taps on an element
  or selects it with the keyboard's Tab key. The styles bound to this state are equivalent to pseudo-class
  `:focus` or `:focus-visible` ones.
- **Pressed**: when the chip component is being activated by the user. When using a mouse, "activation" typically starts when
  the user presses down the primary mouse chip. The styles bound to this state are equivalent to pseudo-class
  `:active` ones.
- **Selected**: Represents its state after interaction. This state is used to indicate which chip component within the multi-select composite widgets is selected.
  This state should be accompanied of `aria-selected` attribute.
- **Activated**: this state is used to indicate which chip component is selected.
  This state should be accompanied of `aria-selected` attribute.
- **Disabled**: this state is used to indicate the chip component is disabled. We should add the `disabled` attribute.

### Color design tokens

The color tokens are based in the color scheme and the state of the chip. So each token usually is a combination of a
color scheme and state (except the backdrop tokens).

There are several pre-defined color schemes: `accent-hight`, `accent`, `neutral`, `blend-base`, `blend-inverse`, `quiet`,
`error`, `success`, `warning`, `help` and `info`.

There are several pre-defined states: `enabled`, `hovered`, `focused`, `pressed`, `expanded`, `selected`, `activated`
`loading`, `loading-error`, `loading-success` and `disabled`.
