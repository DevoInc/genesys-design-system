# Content switcher

The Content switcher allows the user to toggle between different views of related content within the same section of the platform. Only one view is shown at the same time (just one tab activated).




## Overview

### Variants
The Content switcher has two props: size and color scheme (use blend when background color is other than white).

The tabs have the possibility of controlling their status (enabled, active and hover) and deciding if they have an icon or not. The size prop should not be changed (It is controlled from the parent component).

### States
The tab has different states depending on the interaction with the user, with other blocks of the page or with
processes of which he is the trigger.
- **Enable**: It is enable to activate by the user.
- **Hover**: when the user interacts with an element with a pointing device, but does not necessarily activate it.
  It is generally triggered when the user hovers over an element with the cursor (mouse pointer). Sometimes we can use
  this state to highlight an action over other ones. The styles bound to this state are equivalent to pseudo-class
  `:hover` ones.
- **Activated**: this state is used to indicate which tab within single-selection composite widgets is selected.
  This state should be accompanied of `aria-selected` attribute. With this state the HTML tag rendered should not
  be a `button` but a `label`, because a radio should be rendered inside, but not visible.
